 NameDescription
BookmakerId Bookmaker id
ConfigLogLevel Specifies log level threshold used while logging configuration settings used by client (server and local). Possible values are: Debug, Info, Warn, Error, Fatal
ExecutionLogLevel Specifies log level threshold used while logging SDK library code execution. Possible values are: Debug, Info, Warn, Error, Fatal
Logging Indicates whether SDK logging should be enabled
LogPath Root path where SDK logs will be stored (relative or absolute).
MaxThreadPoolThreads Maximum number of thread pool threads
OldLogCleanupInterval Old log clean-up interval.
OldLogMaxAge Old log max-age.
QueueHighWatermark High watermark for all limited priority queue instances (number of items). Signals that a queue will soon reach its full capacity and that we should probably stop processing incoming messages until things calm down a bit.
QueueLowWatermark Low watermark for all limited priority queue instances (number of items). Signals that a queue is getting filled up and that client may experience some additional latency in msg delivery.
QueueMaxCapacity Capacity (max. number of items) that can be stored in any limited priority queue instance.
SafeMode Indicates whether "safe-mode" SDK components are used instead of normal ones.
StatsLogLevel Specifies log level threshold used while logging SDK statistics. Possible values are: Debug, Info, Warn, Error, Fatal
UniquenessStrictness Verify whether we are the only instance running
